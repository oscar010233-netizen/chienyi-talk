# 施工單：老師管理 — 排序欄註明 ＋ 上下按鈕（雙重設計）

出單：Harness 9 ｜ 對象：Codex Engineer 9 ｜ 2026-06-26

## 背景
老師管理 modal 每筆老師右邊有一個裸數字輸入框（`sort_order`），使用者看不懂那個數字是什麼。
老闆要：**保留數字框**（給進階手動填），但**加清楚註明**，並**另外加上下箭頭按鈕**讓一般使用者不用懂數字也能調順序（雙重設計、兩者並存）。

## 動工範圍
**只碰** `components/schedule/TeacherManagerModal.tsx`。
**不要**碰 API（`/api/teachers` 的 PATCH 已支援 `sort_order`，夠用）、不要碰 DB、不要碰其他檔。

## 現況（先讀）
- 清單每筆 row 目前是 grid：`名稱輸入 | sort_order 數字輸入(96px) | 儲存 | 刪除`
- `sort_order` 透過 `handleUpdate`（PATCH name + sort_order）儲存
- 清單顯示順序：API `order by sort_order, name`（數字小的在前）

## 要做的事

### 1. 排序欄位「註明清楚」
- 在 modal 標題副標（現有「只顯示啟用中的老師，刪除會改成封存。」）**下方**或清單**上方**，加一行說明，例如：
  > 排序：數字越小越前面；也可用 ↑ / ↓ 直接調整。
- 數字輸入框旁加一個小 label「排序」（或 placeholder/title 提示），讓使用者一眼知道那格是排序值、不是別的。
- 文案用繁體中文，**務必用 UTF-8 寫入並讀回確認**（避免之前的 mojibake；寫完用 UTF-8 讀回該幾行核對）。

### 2. 加「上 / 下」按鈕（核心）
每筆老師 row 加一組 ↑ / ↓ 按鈕（lucide `ChevronUp` / `ChevronDown`，記得加 import）：
- **↑**：把這位老師往清單上方移一格；清單**第一筆的 ↑ 要 disabled**。
- **↓**：往下移一格；清單**最後一筆的 ↓ 要 disabled**。
- 點擊行為：與上方相鄰那筆**對調位置**，然後把順序**持久化**。

#### 持久化做法（重點，要穩）
因為現有資料可能 `sort_order` 全是 0，單純對調數值看不出差別。請用「**依新位置重新編號**」：
1. 在前端陣列裡把兩筆對調位置。
2. 把整個清單**重新編號** `sort_order = 該筆的新 index`（0,1,2,…）。
3. 對 `sort_order` 實際有變動的老師，各送一次 `PATCH /api/teachers/{id}`（body `{ sort_order }`）。
4. 全部成功後重抓清單（沿用既有 `loadTeachers()`）。
- 移動進行中按鈕要 disabled（沿用 `saving` 旗標即可），避免連點打架。

### 3. 數字框維持可用
- 手動改數字 + 「儲存」的既有流程**保留不動**（進階使用者仍可直接打數字）。
- 上下按鈕與數字框兩種方式並存，調完都以 API 持久化、以重抓清單為準。

### 4. 版面
- row 加了 ↑↓ 後欄位變多，調整 grid 欄寬讓它在 modal 寬度（max-w-xl）下不擠壞；窄螢幕可換行。
- 不要破壞既有 mac 風格（沿用現有 border / rounded / hover）。

## 完工要求
- `npx tsc --noEmit` → 0 error；`npm run lint` → 不得新增 warning。
- dev server 驗：
  1. 清單至少 2 位老師時，點 ↑ / ↓ 順序會改變且重整後仍維持（持久化成功）。
  2. 第一筆 ↑、最後一筆 ↓ 為 disabled。
  3. 手動改數字 + 儲存仍有效。
  4. 註明文字顯示正確（非亂碼）。
  - 測完把測試老師封存清掉。
- 所有新增中文字串：**用 UTF-8 寫，並用 UTF-8 讀回核對**。
- 不要 commit/push（等老闆指示）。
- 回報：改了什麼、四項驗證結果、確認中文無亂碼（附讀回那幾行）。
